## Context

**Current:** `go 1.25.5` in `go.mod`, 30 Go files, ~20 `errors.New` calls, 2 `new(Type)` calls.

**Go 1.26 release notes** (Feb 2026, go.dev/doc/go1.26):

### Language changes
- **`new()` now takes expressions** — `new(yearsSince(born))` replaces `tmp := yearsSince(born); ptr := &tmp`. Huge for JSON serialization optional fields (`Age *int`).
- **Self-referential generics** — `type Adder[A Adder[A]] interface` now legal.

### Tools
- **`go fix` fully revamped** — built on `go vet` framework. Runs same analyzers that produce vet diagnostics. Can apply fixes automatically.
- **`go doc` / `cmd/doc` deleted** — use `go doc` (same flags, same args).
- **`go mod init` defaults to N-1** — 1.26 toolchain creates `go 1.25.0` modules (not 1.26). No effect for us (we're upgrading).

### Runtime
- **Heap base address randomization** — security hardening, build-time `GOEXPERIMENT=norandomizedheapbase64` to disable.
- **Goroutine leak profile** — new `runtime/pprof` profile type, build with `GOEXPERIMENT=goroutineleakprofile`.
- **Green threads / GC** — `GOEXPERIMENT=nogreenteagc` still available, no user-visible API change.

### Standard library
- **`crypto/ecdsa` big.Int fields deprecated** — `PublicKey.X`, `PublicKey.Y`, `PrivateKey.D`. Use new field-based constructors. Does not affect Blacked (no direct ecdsa usage found).
- **`fmt.Errorf` now as cheap as `errors.New`** — allocation parity. Existing `errors.New` calls can stay; new code can prefer `fmt.Errorf` for consistency with wrapping.
- **`errors.AsType[T]`** — new generic `As` variant, type-safe, faster.
- **JPEG encoder/decoder replaced** — bit-exact output may change (not relevant to Blacked).
- **`io.ReadAll` memory optimizations** — already used in colly client.
- **`log/slog` MultiHandler added** — not needed for our use.
- **TLS changes** — 3DES cipher suites dropped, RSA-only key exchange disabled by default, min TLS 1.2. Blacked doesn't control TLS configuration directly (relies on server defaults).
